## 2. 의료 프로토콜 불일치

### 2-1. 화상 냉각 시간 불일치 ⚠️
| 파일 | 냉각 시간 | 온도 조건 |
|------|-----------|-----------|
| `Emergency.jsx` | **20분** (1200초) | 12~25℃ |
| `AnalysisEmergency.jsx` | **15분** (900초) | 15~20℃ |

- 의료 표준은 15~20℃ 흐르는 물에 **20분**이 권장됨.  
- 두 파일을 20분(1200초), 15~25℃로 통일할 것.

### 2-2. CPR 압박 속도 불일치 ⚠️
| 파일 | bpm |
|------|-----|
| `Emergency.jsx` | **110 bpm** (하드코딩) |
| `AnalysisEmergency.jsx` CPRMetronome | **120 bpm** |

- 표준(AHA): 100~120 bpm. 두 화면을 120 bpm으로 통일 권장.
- `Emergency.jsx:171`: `const [bpm] = useState(110)` → 120으로 변경 필요.

### 2-3. 트리아지(의식 수준) 판별 결과 매핑 오류 ⚠️
- **위치**: `Emergency.jsx:308~312`
- **문제**: "부르면 대답을 하나요?" (언어 반응 있음) → `골절 / 탈구` 로 연결됨.  
  언어 반응이 있는 환자에게 골절 처치 가이드를 보여주는 것은 의료적으로 부적절함.
- **제안**: "부르면 대답을 하나요?" → 일반 외상/상태 파악 가이드 또는 개방형 문진 화면으로 연결.

## 3. 기능 누락 (UX/안전)

### 3-1. 환자 알레르기 경고 미표시
- `AnalysisEmergency.jsx`는 아스피린 알레르기를 체크하여 경고를 표시하지만, `Emergency.jsx`의 심폐소생술/심근경색 가이드에는 환자 알레르기 정보가 표시되지 않음.
- 약물 투여가 필요한 프로토콜에서는 `patient.allergies`를 확인하여 경고 배너를 표시해야 함.

### 3-2. 세션 데이터 미지속
- 응급 처치 로그(`sessionLogs`)가 `useState` 로컬 상태에만 저장됨.
- 페이지 이탈/새로고침 시 모든 기록 소실.
- **제안**: `localStorage` 혹은 서버 동기화로 응급처치 세션을 지속 저장.

### 3-3. AI 후속 지침이 고정 텍스트
- **위치**: `Emergency.jsx:388~390` (SUMMARY 화면)
- 실제 수행한 처치 내용과 관계없이 항상 동일한 3개 문장이 표시됨.
- **제안**: `activeAction` 기준으로 처치별 맞춤 후속 지침 표시.

### 3-4. 단계 완료 취소 기능 없음
- 단계를 완료 처리(`completedSteps`)하면 되돌릴 수 없음.
- 잘못 클릭한 경우 세션을 초기화해야 함.
- **제안**: 완료된 단계 재클릭 시 취소 가능하도록 토글 처리.

### 3-5. AnalysisEmergency — 환자 실데이터 미연동
- `RISK_SCORE`, `DIAGNOSES`, `VITALS`, `TREND` 가 모두 컴포넌트 바깥에 상수로 정의됨.
- `patient` prop을 받아도 실제 바이탈 수치에 반영되지 않음.
- **제안**: patient prop의 실측 데이터를 VITALS/TREND 에 연동.
